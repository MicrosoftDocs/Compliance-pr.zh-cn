## <a name="how-does-microsoft-365-ensure-isolation-between-customer-tenants"></a><span data-ttu-id="5f39b-114">Microsoft 365 如何确保客户租户之间的隔离？</span><span class="sxs-lookup"><span data-stu-id="5f39b-114">How does Microsoft 365 ensure isolation between customer tenants?</span></span>

<span data-ttu-id="5f39b-115">Microsoft 的云服务在假定所有租户都可能会与所有其他租户建立恶意。</span><span class="sxs-lookup"><span data-stu-id="5f39b-115">Microsoft's cloud services are built on the assumption that all tenants are potentially hostile to all other tenants.</span></span> <span data-ttu-id="5f39b-116">为了正确地将租户相互隔离，Microsoft 实现了各种隔离技术和控制。</span><span class="sxs-lookup"><span data-stu-id="5f39b-116">To properly isolate tenants from one another, Microsoft implements a variety of isolation technologies and controls.</span></span> <span data-ttu-id="5f39b-117">这些控制措施旨在防止信息泄露或未经授权访问跨租户的客户数据，并防止一个租户的操作对另一个租户的服务产生不利影响。</span><span class="sxs-lookup"><span data-stu-id="5f39b-117">These controls are designed to safeguard against information leakage or unauthorized access to customer data across tenants and to prevent the actions of one tenant from adversely affecting the service for another tenant.</span></span>

<span data-ttu-id="5f39b-118">在 Microsoft 365 租户中，使用 Azure Active Directory (Azure AD) 以逻辑方式隔离客户内容。</span><span class="sxs-lookup"><span data-stu-id="5f39b-118">Customer content is logically isolated within Microsoft 365 tenants using Azure Active Directory (Azure AD).</span></span> <span data-ttu-id="5f39b-119">Microsoft 365 中的用户身份验证不仅验证用户身份，而且还验证用户帐户所属的租户标识，从而阻止用户访问其租户环境外部的数据。</span><span class="sxs-lookup"><span data-stu-id="5f39b-119">User authentication in Microsoft 365 verifies not only the user identity, but also the tenant identity the user account is part of, preventing users from accessing data outside their tenant environment.</span></span> <span data-ttu-id="5f39b-120">若要对 Azure AD 的逻辑隔离进行补充，客户内容将始终在 rest 和途进行加密。</span><span class="sxs-lookup"><span data-stu-id="5f39b-120">To supplement the logical isolation of Azure AD, customer content is always encrypted at rest and in transit.</span></span> <span data-ttu-id="5f39b-121">单个服务还可以提供其他层的租户隔离，例如在单独的加密数据库中对租户数据隔离的 SharePoint Online。</span><span class="sxs-lookup"><span data-stu-id="5f39b-121">Individual services may also provide additional layers of tenant isolation, such as SharePoint Online isolation of tenant data in separate, encrypted databases.</span></span>

## <a name="how-does-microsoft-365-engineer-resilient-services-that-avoid-single-points-of-failure"></a><span data-ttu-id="5f39b-122">如何避免单一故障点的 Microsoft 365 工程师弹性服务？</span><span class="sxs-lookup"><span data-stu-id="5f39b-122">How does Microsoft 365 engineer resilient services that avoid single points of failure?</span></span>

<span data-ttu-id="5f39b-123">Microsoft 设计和构建云服务，以最大限度地提高可靠性并最大限度地减少对正常操作的故障和挑战的客户面临的负面影响。</span><span class="sxs-lookup"><span data-stu-id="5f39b-123">Microsoft designs and builds cloud services to maximize reliability and minimize the negative effects on customers in the face of faults and challenges to normal operations.</span></span> <span data-ttu-id="5f39b-124">此策略首先是连接地理分散的数据中心的网络的设计。</span><span class="sxs-lookup"><span data-stu-id="5f39b-124">This strategy begins with the design of the network connecting our geographically distributed datacenters.</span></span> <span data-ttu-id="5f39b-125">Microsoft 的网络体系结构包括直接互连和多个网络路径。</span><span class="sxs-lookup"><span data-stu-id="5f39b-125">Microsoft's network architecture includes direct interconnections and multiple network paths.</span></span> <span data-ttu-id="5f39b-126">Microsoft 365 服务利用此冗余来自动路由失败的流量，以提高服务质量。</span><span class="sxs-lookup"><span data-stu-id="5f39b-126">Microsoft 365 services leverage this redundancy to automatically route traffic around failures to improve service quality.</span></span>

<span data-ttu-id="5f39b-127">在服务级别，Microsoft 365 的恢复策略优先于软件弹性。</span><span class="sxs-lookup"><span data-stu-id="5f39b-127">At the service level, Microsoft 365's resilience strategy prioritizes software resiliency.</span></span> <span data-ttu-id="5f39b-128">只要有可能，我们的服务将通过自动服务运行状况监视部署到主动/主动配置中，从而使服务能够在无人干预的情况下检测和恢复许多常见故障和故障。</span><span class="sxs-lookup"><span data-stu-id="5f39b-128">Wherever possible, our services are deployed in active/active configurations with automated service health monitoring, allowing the service to detect and recover from many common faults and failures without human intervention.</span></span> <span data-ttu-id="5f39b-129">除了主动/主动配置之外，Microsoft 365 服务可确保服务在单独的故障区域中部署，从而防止一个区域中的故障影响其他区域的可用性，从而提高了容错能力。</span><span class="sxs-lookup"><span data-stu-id="5f39b-129">In addition to active/active configurations, Microsoft 365 services increase fault-tolerance by ensuring the service is deployed in separate fault zones, preventing a fault in one zone from affecting the availability of other zones.</span></span>

<span data-ttu-id="5f39b-130">数据恢复通过保护 Microsoft 365 服务中的数据的完整性和可用性来补充服务弹性。</span><span class="sxs-lookup"><span data-stu-id="5f39b-130">Data resiliency complements service resiliency by protecting the integrity and availability of data in Microsoft 365 services.</span></span> <span data-ttu-id="5f39b-131">我们的服务使用本地存储冗余和地域冗余将客户数据的副本复制到不同的故障区域中。</span><span class="sxs-lookup"><span data-stu-id="5f39b-131">Our services use local storage redundancy and geo-redundancy to replicate copies of customer data into different fault zones.</span></span> <span data-ttu-id="5f39b-132">如果一个故障区域中的数据已损坏或丢失，则可以在另一个容错区域中访问它，而不会丢失可用性。</span><span class="sxs-lookup"><span data-stu-id="5f39b-132">If data is corrupted or lost in one fault zone, it can be accessed in another fault zone without loss of availability.</span></span> <span data-ttu-id="5f39b-133">自动完整性检查会自动恢复受多种物理或逻辑损坏影响的数据。</span><span class="sxs-lookup"><span data-stu-id="5f39b-133">Automated integrity checking automatically restores data impacted by many kinds of physical or logical corruption.</span></span> <span data-ttu-id="5f39b-134">Microsoft 365 还为客户提供了用于在 Exchange Online 和 SharePoint Online 中恢复客户意外删除或修改的数据的工具。</span><span class="sxs-lookup"><span data-stu-id="5f39b-134">Microsoft 365 also provides customers with tools to restore data accidentally deleted or modified by the customer in Exchange Online and SharePoint Online.</span></span>

<span data-ttu-id="5f39b-140">Microsoft 365 体系结构将定期进行验证，并使用基于云的工具自动进行验证，以验证与安全原则的协调并持续测试隔离和恢复功能。</span><span class="sxs-lookup"><span data-stu-id="5f39b-140">Microsoft 365 architecture is validated regularly and automatically using cloud-based tools to verify alignment with our security principles and to continuously test isolation and resiliency features.</span></span> <span data-ttu-id="5f39b-141">体系结构验证可用于自动识别服务的当前状态从所需状态偏差的实例，并标记所有用于检查和缓解的偏差。</span><span class="sxs-lookup"><span data-stu-id="5f39b-141">Architectural validation works to automatically identify instances where the current state of the service has drifted from the desired state, flagging any deviations for review and mitigation.</span></span> <span data-ttu-id="5f39b-142">体系结构验证的目标是确保服务基础结构的安全功能继续按预期运行。</span><span class="sxs-lookup"><span data-stu-id="5f39b-142">The goal of architecture validation is to ensure the security capabilities of our service infrastructure continue to function as expected.</span></span>
