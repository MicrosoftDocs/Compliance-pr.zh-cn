## <a name="what-role-does-encryption-play-in-protecting-customer-content"></a><span data-ttu-id="be451-104">加密在保护客户内容中扮演什么角色？</span><span class="sxs-lookup"><span data-stu-id="be451-104">What role does encryption play in protecting customer content?</span></span>

<span data-ttu-id="be451-105">大多数 Microsoft 商业云服务都是多租户的，这意味着客户内容可以存储在与其他客户相同的物理硬件上。</span><span class="sxs-lookup"><span data-stu-id="be451-105">Most Microsoft business cloud services are multitenant, meaning that customer content may be stored on the same physical hardware as that of other customers.</span></span> <span data-ttu-id="be451-106">为了保护客户内容的机密性，Microsoft 365 使用一些最强和最安全的加密协议来加密静态和传输中的所有数据。</span><span class="sxs-lookup"><span data-stu-id="be451-106">To protect the confidentiality of customer content, Microsoft 365 encrypts all data at rest and in transit with some of the strongest and most secure encryption protocols available.</span></span>

<span data-ttu-id="be451-107">加密不是强大的访问控制的替代方案。</span><span class="sxs-lookup"><span data-stu-id="be451-107">Encryption is not a substitute for strong access controls.</span></span> <span data-ttu-id="be451-108">Microsoft 365 的访问控制策略 of Zero ZSA Access (的) 保护客户内容免受 Microsoft 员工的未经授权的访问。</span><span class="sxs-lookup"><span data-stu-id="be451-108">Microsoft 365's access control policy of Zero Standing Access (ZSA) protects customer content from unauthorized access by Microsoft employees.</span></span> <span data-ttu-id="be451-109">加密通过在 Microsoft 365 系统之间或 Microsoft 365 与客户之间进行传输时防止内容被阅读，来保护客户内容的机密性，从而补充了访问控制。</span><span class="sxs-lookup"><span data-stu-id="be451-109">Encryption complements access control by protecting the confidentiality of customer content wherever it is stored and by preventing content from being read while in transit between Microsoft 365 systems or between Microsoft 365 and the customer.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-at-rest"></a><span data-ttu-id="be451-110">Microsoft 365 如何对数据进行静态加密？</span><span class="sxs-lookup"><span data-stu-id="be451-110">How does Microsoft 365 encrypt data-at-rest?</span></span>

<span data-ttu-id="be451-111">Microsoft 365 中的所有客户内容都受一个或多个加密形式的保护。</span><span class="sxs-lookup"><span data-stu-id="be451-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="be451-112">Microsoft 服务器使用 BitLocker 在卷级加密包含客户内容的磁盘驱动器。</span><span class="sxs-lookup"><span data-stu-id="be451-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="be451-113">BitLocker 提供的加密可在其他进程或控件发生失误时保护客户内容 (例如，对硬件) 的访问控制或回收，从而导致对包含客户内容的磁盘的未经授权的物理访问。</span><span class="sxs-lookup"><span data-stu-id="be451-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="be451-114">Exchange Online、Microsoft 团队、SharePoint Online 和 OneDrive for Business 也会在应用程序层使用服务加密来加密客户内容。</span><span class="sxs-lookup"><span data-stu-id="be451-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="be451-115">服务加密在强加密保护的顶层提供了权限保护和管理功能。</span><span class="sxs-lookup"><span data-stu-id="be451-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="be451-116">它还允许在 Windows 操作系统和由这些操作系统存储或处理的客户数据之间进行分离。</span><span class="sxs-lookup"><span data-stu-id="be451-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="be451-117">Microsoft 365 如何加密数据传输？</span><span class="sxs-lookup"><span data-stu-id="be451-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="be451-118">Microsoft 365 产品和服务使用强大的传输协议（如 TLS）防止未经授权的参与方在客户数据通过网络移动时窃听。</span><span class="sxs-lookup"><span data-stu-id="be451-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="be451-119">传输中的数据示例包括正在传递的邮件、联机会议中发生的对话或在数据中心之间复制的文件。</span><span class="sxs-lookup"><span data-stu-id="be451-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="be451-120">在 Microsoft 365 中，只要用户的设备与 Microsoft 服务器通信或 Microsoft 服务器与另一台服务器通信，数据就会被视为 "在传输中"。</span><span class="sxs-lookup"><span data-stu-id="be451-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="be451-121">Exchange Online、SharePoint Online、Microsoft 团队和 Office Online 都使用 TLS，以确保数据在传输过程中保持机密。</span><span class="sxs-lookup"><span data-stu-id="be451-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="be451-122">Microsoft 365 如何管理用于加密的密钥？</span><span class="sxs-lookup"><span data-stu-id="be451-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="be451-123">强加密只与用于加密数据的密钥一样安全。</span><span class="sxs-lookup"><span data-stu-id="be451-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="be451-124">Microsoft 使用其自己的安全证书对 TLS 连接进行数据传输。</span><span class="sxs-lookup"><span data-stu-id="be451-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="be451-125">对于静态数据，受 BitLocker 保护的卷使用完整的卷加密密钥进行加密，该密钥使用批量主密钥进行加密，而批量主密钥又绑定到服务器中 (TPM) 的受信任的平台模块。</span><span class="sxs-lookup"><span data-stu-id="be451-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="be451-126">BitLocker 使用符合 FIPS 标准的算法，以确保不会以明文形式在线路中存储或发送加密密钥。</span><span class="sxs-lookup"><span data-stu-id="be451-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="be451-127">服务加密为 Exchange Online、Microsoft 团队、SharePoint Online 和 OneDrive for Business 中的客户静态数据提供了额外的加密层。</span><span class="sxs-lookup"><span data-stu-id="be451-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="be451-128">服务加密为客户提供了两个加密密钥管理选项： Microsoft-托管密钥或客户密钥。</span><span class="sxs-lookup"><span data-stu-id="be451-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="be451-129">使用 Microsoft 管理的密钥时，Microsoft 365 服务会自动生成并安全地存储用于服务加密的根密钥。</span><span class="sxs-lookup"><span data-stu-id="be451-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="be451-130">如果客户具有控制其自己的根加密密钥的要求，则可以利用客户密钥进行服务加密。</span><span class="sxs-lookup"><span data-stu-id="be451-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="be451-131">使用客户密钥，客户可以使用本地硬件服务模块 (HSM) 或 Azure Key Vault (AKV) 生成自己的加密密钥。</span><span class="sxs-lookup"><span data-stu-id="be451-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="be451-132">客户根密钥存储在 AKV 中，在这些密钥中，它们可用作加密客户邮箱数据或文件的 keychains 之一的根。</span><span class="sxs-lookup"><span data-stu-id="be451-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="be451-133">客户根键只能由 Microsoft 365 服务代码间接访问以进行数据加密，Microsoft 员工无法直接访问它们。</span><span class="sxs-lookup"><span data-stu-id="be451-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
